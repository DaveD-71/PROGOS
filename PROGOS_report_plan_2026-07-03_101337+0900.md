Build the report around an explicit evidence base and review method at the front.

Include:

- scope of review
- files reviewed
- what was and was not examined
- distinction between direct evidence, inference, and limitation
- warning that no unpublished psychometric or vendor-internal documentation was available

Use the repo evidence explicitly:

- PROGOS result PDFs for Aozawa and Saki

- CEFR workbook
- completed CEFR comparison report for Aozawa

### 2. Core report structure

Use the broad structure already captured in `PROGOS_test_evaluation_report_structure.md`, but tighten it so every section is evidence-led.

Planned sections:

- Executive Summary
- Evidence Base and Review Method
- Test Purpose and Observed Design
- Task and Construct Analysis
- CEFR Alignment and Score-Report Analysis
- Reliability and Interpretation Limits
- Fairness, Accessibility, and Practicality
- Legal and Data-Governance Observations
- Suitability for Organisational Uses
- Limitations of This Review
- Conclusions and Recommendations

### 3. Test design analysis

Describe only what can be supported from screenshots and reports.

Include:

- observed task types
- visible timing and prompt conditions where captured
- distinction between spoken production and simulated interaction
- explicit finding that the “communication activity” is not live two-way interaction evidence

Avoid:

- guessing task counts or admin rules not evidenced
- claiming full methodology details that are not in the repo

### 4. CEFR and report-comment analysis

Promote the existing CEFR comparison work into the main report.

Include:

- overall assessment is not a CEFR competence and should not be matched
- six individual competences only
- same-competence same-level CEFR comparisons
- confirmed Yes/No match findings:
  - Range: No
  - Accuracy: Yes
  - Fluency: Yes
  - Interaction: No
  - Coherence: Yes
  - Phonology: Yes
- for the two No cases, explain both the official same-scale descriptor and the closest wording match

Also analyse:

- whether the score reports appear templated across Aozawa and Saki
- whether advice/examples are level-based rather than performance-specific
- how much diagnostic value the reports actually provide

### 5. Legal and governance section

Base this section on the newly captured live pages, not only the old chat discussion.

Use:

- `2026-07-03_progos_kiyaku_text.md`

### 6. Organisational suitability section

Evaluate PROGOS against the organisation’s stated needs without overstating the comparison.

Use:

- broad benchmarking
- placement support
- pre/post progress measurement
- course design
- teacher assignment
- diagnostic use

For each use case:

- state the likely suitability level
- tie the judgement to evidence already established in the report
- avoid claiming sensitivity, reliability, or invalidity beyond the available evidence
